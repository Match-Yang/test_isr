## Version 4.17.0

**Release Date: 2025-05-29**

<h5>New Features</h5>

| <div style={{width:"130px"}}>Feature</div> |Description | <div style={{width:"200px"}}>Related APIs</div> |
| :------------------- | :----------- | :----------------------------------- |
| iOS PIP | CallKit supports iOS PIP by default. To disable the feature, you can set `ZegoCallInvitationConfig.pip.iOS.support` or `ZegoUIKitPrebuiltCallConfig.pip.iOS.support` to `false`. | <ul><li>[ZegoCallInvitationConfig.pip.iOS.support](https://pub.dev/documentation/zego_uikit_prebuilt_call/latest/zego_uikit_prebuilt_call/ZegoCallInvitationPIPIOSConfig/support.html)</li><li>[ZegoUIKitPrebuiltCallConfig.pip.iOS.support](https://pub.dev/documentation/zego_uikit_prebuilt_call/latest/zego_uikit_prebuilt_call/ZegoCallPIPIOSConfig/support.html)</li></ul> |
| Delaying Auto-entry into Offline Calls After App Launch | To prevent conflicts between your app’s startup logic and call interface navigation, set `autoEnterAcceptedOfflineCall` to false. This lets your app complete tasks like navigation redirection, cache loading, or message fetching before manually calling `enterAcceptedOfflineCall` to join the call. | <ul><li>[ZegoCallInvitationOfflineConfig > autoEnterAcceptedOfflineCall](https://pub.dev/documentation/zego_uikit_prebuilt_call/latest/zego_uikit_prebuilt_call/ZegoCallInvitationOfflineConfig/autoEnterAcceptedOfflineCall.html)</li><li>[enterAcceptedOfflineCall](https://pub.dev/documentation/zego_uikit_prebuilt_call/latest/zego_uikit_prebuilt_call/ZegoUIKitPrebuiltCallInvitationService/enterAcceptedOfflineCall.html)</li></ul> |
| Offline Call Acceptance (Locked Screen) | Android users can accept offline call invitations via CallKit even when the screen is locked. | – |

<h5>Enhancements</h5>

- Modified the notification pop-up method when CallKit is in the Android background.
- Compatible with the offline logic of ZIMKit .
- Improved the internal minimized interface within CallKit.

<h5>Bug Fixes</h5>

- Fixed known issues.
