<Steps>
<Step title="配置回调地址">
请联系 ZEGO 技术支持配置用于接收异常事件的回调地址。
</Step>
<Step title="启用回调接收">
[创建智能体实例](./../API%20reference/Agent%20Instance%20Management/Create%20Agent%20instance.mdx) 时，请求参数 `CallbackConfig.Interrupted` 设置为 1。
</Step>
<Step title="接收回调">
当智能体被打断时，会回调事件通知到配置的回调地址，内容示例如下：
```json {13}
{
    "AppId": 123456789,
    "Nonce": "abcdd22113",
    "Timestamp": 1747033950524，
    "Sequence": 1921825797275873300,
    "Signature": "XXXXXXX",
    "Event": "Interrupted",
    "RoomId": "90000001237",
    "AgentInstanceId": "1921825671047294976",
    "AgentUserId": "apitest689_agent",
    "Data": {
        "Round": 1481651956,
        "Reason": 1
    }
}
```
`Reason`（被打断的原因）解释如下：
| 参数   | 类型   | 说明         |
|--------|--------|--------------|
| Reason | Int32  | 被打断的原因：<ul><li>1：用户讲话。</li><li>2：您在服务端 [主动调用 LLM](./../API%20reference/Agent%20Instance%20Control/Send%20Agent%20instance%20LLM.mdx)。</li><li>3：您在服务端 [主动调用 TTS](./../API%20reference/Agent%20Instance%20Control/Send%20Agent%20instance%20TTS.mdx)。</li><li>4：您在服务端 [打断智能体实例](./../API%20reference/Agent%20Instance%20Control/Interrupt%20Agent%20Instance.mdx)。</li></ul> |
</Step>
</Steps>
