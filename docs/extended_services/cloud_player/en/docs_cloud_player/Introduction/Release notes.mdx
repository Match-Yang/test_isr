## November 2023

<table>
  
<tbody><tr>
<th>Release Item</th>
<th>Release Description</th>
<th>Related Documents</th>
<th>Release Date</th>
</tr>
<tr>
<td rowspan="2">Server Update</td>
<td>When creating a cloud player, a new parameter <code>RoomUserUpdate</code> is added.When this parameter is set to <code>1</code>, the <a href="/article/api?doc=Express_Video_SDK_API~java_android~class~IZegoEventHandler#on-room-user-update" target="_blank">onRoomUserUpdate</a> callback will be triggered on the client side for rooms created by the cloud player. This callback notifies the developer when other users come online or go offline in the room, causing changes in the user list.</td>
<td>[Create a cloud player](/cloud-player-server/create-player)</td>
<td rowspan="2">November 14, 2023</td>
</tr>
<tr>
<td>A new callback event type (the <code>EventType</code> parameter adds a new value <code>4</code>) is added, which represents abnormal events detected by the cloud player during runtime.After detecting such abnormal events, the cloud player will automatically retry internally. If multiple callbacks of this type of exception are received within a short period of time, developers need to check if the resource server access is normal or perform other fault-tolerant processing.</td>
<td>[Related callbacks and verification instructions](/cloud-player-server/verification-instructions)</td>
</tr>
